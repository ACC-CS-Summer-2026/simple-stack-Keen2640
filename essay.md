## 1. Explain your AI usage on this assignment, what you used, and why.

I used Claude to help me complete this assignment. ## 2. Explain and give an example of at least one error AI made that you caught and corrected.

One mistake Claude made was suggesting additional public methods for the stack class that were not allowed by the assignment specifications. After reviewing the assignment instructions, I noticed that only `push`, `pop`, `peek`, and `isEmpty` were supposed to be public. I corrected the design and removed the unnecessary methods so that the class matched the required interface.

## 3. Explain and give an example of at least one instance of AI code that was correct but you could improve it and did so.

Claude generated code that correctly checked whether the stack was empty before performing a `pop` operation. Although the code worked, I improved it by making the error handling consistent between both `pop` and `peek`. I also improved readability by using constants instead of hard-coded values and organizing the files according to the required architecture.

## 4. Explain and give an example of one thing you learned by using AI, some technique or concept you didn't know or understand before and now you do.

Using Claude helped me better understand abstract data types and how a stack works internally. I learned about stack overflow and underflow conditions and how they should be handled. I also learned why exception handling or pass-by-reference can be used when an integer return value alone is not enough to indicate an error.

## 5. Based on your experience in this assignment, do you think you could have done better without AI? Explain your answer.

No, I do not think I would have done better without AI. Claude helped me understand the assignment requirements and solve problems more efficiently. However, I still had to review the code, test it thoroughly, and make sure it followed the specifications. AI was a useful tool, but I was responsible for understanding and verifying the final solution.
